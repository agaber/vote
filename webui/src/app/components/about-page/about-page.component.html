About this app

Voting

Instant-runoff voting (or ranked-choice voting or alternate vote) is a way of counting votes in single-winner elections. In such elections, voters can choose multiple candidates in order of preference. Behind the scenes, the algorithm can calculate a winner over multiple rounds if necessary, doing its best to keep voter preferences in mind.

CGP Grey's series of YouTube videos do a great job of explaining the problems with traditional first-past-the-post elections and how other voting algorithms can yield fairer and more democratic results.

This App

The original intent of this application was to support instant run-off elections. Users can create random elections with an arbitrary number of choices and let people vote by submitting their ranked preferences and a winner can be determined using the instant run-off algorithm.

Truth be told however, an ulterior motive for creating this app was for me (agaber) to experiment with a few technologies. What follows are a few highlights of the relevant technologies used to build this app.

  • Kubernetes

    I am using Kubernetes to manage microservices on the cloud. For now the cloud provider I'm currently using is DigitalOcean.

  • Java with Spring Boot

    So I actually like the Java programming language (I know right?) but until now I have never tried Spring so I thought I'd give it a shot. To be completely honest, I don't love it because there are quite a few things that seem to happen at runtime or hidden behavior that changes based on the binaries you import, but I was able to get set up quickly and I was able to focus most of my time on writing code instead of setting up boilerplate which was very cool.

  • Mongo DB

    To be honest any old database would've sufficed when I got started but MongoDB is very easy to set up and their Atlas platform makes it cheap and easy to have something running in the cloud. For now I don't need any of the features provided by a relational database and since MongoDB is part of the NoSQL family of databases, it can scale to handle large volumes of data with high read and write throughput.

  • Angular + TypeScript

    Angular is a soup to nuts web development framework well suited for large projects and enterprise level efforts. It provides developers with a lot right out of the box which creates a bit of a learning curve initially but it also means we automatically get preprocessors and task runners that enable us to use things like SCSS and TypeScript and production grade deploy binaries.

The source code in its entirity can be found on GitHub.
